FIRMA DE ZEROONE PARA WINDOWS
=============================

Qué hace la firma
-----------------
La firma de código demuestra que el ejecutable y el instalador pertenecen al
publicador del certificado y que no fueron modificados después de compilarlos.
El nombre que Windows muestra como "Editor" será el nombre legal incluido en
el certificado, no el campo author de package.json.

Qué se necesita
---------------
1. Un certificado RSA de Code Signing emitido por una autoridad de confianza,
   o una cuenta configurada de Microsoft Trusted Signing.
2. Para el método PFX: el archivo .pfx/.p12 y su contraseña.
3. Para publicar en GitHub: un GH_TOKEN con permiso para crear la release.

Un certificado autofirmado sirve únicamente para pruebas internas. En equipos
de clientes seguirá apareciendo como no confiable, salvo que instalen
manualmente ese certificado como raíz de confianza.

Compilar con un PFX sin publicar
--------------------------------
Abrí PowerShell en la carpeta del proyecto y configurá las variables solo para
esa terminal:

  $env:CSC_LINK = "C:\ruta-segura\certificado-zeroone.pfx"
  $env:CSC_KEY_PASSWORD = "CONTRASEÑA_DEL_CERTIFICADO"
  npm run build:signed

Al terminar, borrá la contraseña de la sesión:

  Remove-Item Env:CSC_KEY_PASSWORD -ErrorAction SilentlyContinue
  Remove-Item Env:CSC_LINK -ErrorAction SilentlyContinue

Compilar, firmar y publicar la actualización
---------------------------------------------
Además de las dos variables anteriores:

  $env:GH_TOKEN = "TOKEN_DE_GITHUB"
  npm run release

El comando release exige una firma válida. Si falta el certificado o la
contraseña, la compilación se detendrá en vez de publicar archivos sin firma.

Nunca subas a GitHub el PFX, la contraseña ni el token. El proyecto ignora
*.pfx, *.p12, *.cer y la carpeta certificates/ como protección adicional.

Comprobar la firma
------------------
Después de compilar, ejecutá:

  Get-AuthenticodeSignature ".\dist\win-unpacked\ZeroOne.exe" |
    Format-List Status, StatusMessage, SignerCertificate

  $instalador = Get-ChildItem ".\dist\ZeroOne-Setup-*.exe" |
    Sort-Object LastWriteTime | Select-Object -Last 1
  Get-AuthenticodeSignature $instalador.FullName |
    Format-List Status, StatusMessage, SignerCertificate

El campo Status debe mostrar Valid en ambos archivos.

Compatibilidad de esta transición
---------------------------------
La aplicación ya se presenta como ZeroOne, pero conserva internamente el appId,
la carpeta de sesiones y el repositorio de actualizaciones anteriores. Eso
permite que la actualización reemplace la instalación existente y que las
líneas, Google Contacts y la IA local sigan disponibles.
